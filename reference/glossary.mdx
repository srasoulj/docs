A short dictionary of terms you'll see in the docs, your workspace, or
while using a Rumizi menu.

## Product terms

| Term | Meaning |
| --- | --- |
| **Venue** | A restaurant, café, or bar running on Rumizi. One venue = one workspace. |
| **Workspace** | The vendor side of Rumizi — where owners and staff edit their menu and settings. Lives at `rumizi.com/your-venue/w`. |
| **Public menu** | The diner-facing menu at `rumizi.com/your-venue`. What people scan into. |
| **QR code** | The scannable code on the table that opens the public menu. |
| **Menu** | A collection of categories and items (for example *Lunch*, *Dinner*, *Drinks*). A venue can have more than one. |
| **Category** | A section of a menu: *Starters*, *Mains*, *Desserts*. |
| **Item** | A single dish or drink. |
| **Modifier / option group** | A choice attached to an item — *Size: S/M/L*, *Spice: mild/medium/hot*. |
| **Tag** | A short label on an item: *Vegetarian*, *Spicy*, *Chef's pick*. Diners can filter by tag. |
| **Featured** | An item pinned to the *Popular* carousel at the top of the menu. |
| **Cross-sell** | A suggested pairing that appears on the item detail page (*"Goes well with"*). |

## Ordering and carts

| Term | Meaning |
| --- | --- |
| **Cart / draft** | The list of items a diner has picked on a given device. Stored on the phone, not shared unless a group is started. |
| **Order together** | Shared-cart mode where multiple diners at one table add to the same cart. |
| **Group cart / GROUP draft** | The shared cart that *Order together* creates. |
| **Share token** | The secret code in an *Order together* invite link. |
| **Participant** | A diner attached to a group cart. |

## Publishing

| Term | Meaning |
| --- | --- |
| **Working copy** | What you're editing in the vendor workspace. Not visible to diners yet. |
| **Published copy / snapshot** | The frozen version of the menu diners actually see. Created when you tap **Publish**. |
| **Publish** | The action that takes the current working copy and makes it the live menu. |
| **Revert last publish** | Roll back to the previous published snapshot. |
| **Preview** | Viewing the working copy on the public URL without publishing. Appends `?menu=<menu-id>` while signed in. |

## Availability

| Term | Meaning |
| --- | --- |
| **In stock** | The item is available. |
| **Out of stock / 86** | The item is temporarily unavailable. Still shown to diners but dimmed. |
| **Out until tomorrow** | Automatic return at midnight in the venue's timezone. |
| **Out until next opening** | Automatic return when the next scheduled window starts. |
| **Schedule / open hours** | Time windows when a menu is considered open. Used to pick the default menu at a given time and to power *Out until next opening*. |

## Languages and translation

| Term | Meaning |
| --- | --- |
| **Default language** | The venue's primary language. Used when a translation is missing. |
| **Supported languages** | The languages a venue enables for its diners. |
| **Language switcher** | The control in the menu header diners use to change language. |
| **Locale** | The code for a language (e.g. `en`, `es`, `fa`). |

## Media

| Term | Meaning |
| --- | --- |
| **Gallery** | The venue's shared media library for item, category, and landing photos. |
| **AI image** | An image generated or improved by AI from your item details. |
| **Hero image** | The full-width banner at the top of a venue's landing page. |

## Accounts and access

| Term | Meaning |
| --- | --- |
| **Owner** | Top-level role on a venue. Can do everything, including billing and staff changes. |
| **Staff** | Day-to-day role on a venue. Can edit menus, change stock, run schedules. Can't manage billing or other staff. |
| **Customer account** | An optional diner account scoped to a single venue (for posting in that venue's community). |
| **Magic link** | A one-time sign-in link Rumizi emails to vendors instead of using passwords. |

## Billing

| Term | Meaning |
| --- | --- |
| **AI credit** | Prepaid balance a venue spends on AI usage (menu extraction, assistants, AI images). The software itself is free. |
| **Free credit** | The $20 of AI credit every new venue gets at signup, with no time limit. |
| **Top-up** | A one-off purchase that adds AI credit to a venue's balance. Any amount from $1 to $10,000. |
| **OpenRouter** | The AI gateway Rumizi routes requests through. AI usage is billed at the OpenRouter cost plus a flat 25% markup. |
| **Stripe** | The billing provider Rumizi uses for card payments on top-ups. |
